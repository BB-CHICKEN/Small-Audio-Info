# MoeKoe Music 音乐信息显示插件（增强版）

[![Version](https://img.shields.io/badge/version-1.5.0-blue)]()
[![License](https://img.shields.io/badge/license-MIT-green)]()
[![MoeKoe](https://img.shields.io/badge/MoeKoe-%E2%89%A51.6.5-ff69b4)]()

> 在 MoeKoe Music 播放器右下角实时显示当前歌曲的音频采样率、位深度与文件大小，快速了解音乐文件质量与体积。

---

## 📖 简介

本插件专为 **MoeKoe Music 桌面版**（基于 Electron）开发，通过**直接解析音频文件头部**（仅读取前 16KB 数据）获取信息，无需额外网络请求，兼容本地文件与网络流，响应迅速且准确。

与旧版依赖 API 不同，本插件采用纯前端解析方案，轻量、高效、零依赖。

---

## ✨ 功能特点

| 特性 | 说明 |
|------|------|
| 🎵 **智能解析** | 自动识别 FLAC 和 MP3 格式，提取采样率与位深度（FLAC 支持位深度解析，MP3 提供采样率） |
| 📡 **无 API 依赖** | 不再依赖 MoeKoe Music 内部 HTTP API，直接从歌曲 URL 发起 Range 请求 |
| 📍 **位置固定** | 显示在播放器容器 `.player-container` 右下角，不干扰主要控制按钮 |
| 🔄 **实时更新** | 每 1 秒自动检测当前歌曲变化，切歌后立即刷新信息 |
| 🛡️ **容错性强** | 解析失败时显示"未知"或"加载中…"，不影响播放器正常运行 |
| 🚀 **轻量高效** | 仅读取 16KB 文件头部数据，极低带宽消耗 |

## 📦 安装方法

### 方法一：通过插件管理安装（推荐）

1. 下载插件压缩包（`.zip`）至本地
2. 打开 **MoeKoe Music** → **设置** → **插件管理**
3. 点击 **"安装插件"**，选择下载的 `.zip` 文件
4. 安装完成后**重启 MoeKoe Music**，插件即自动生效

### 方法二：手动加载源码

1. 将源码克隆或下载到本地文件夹
2. 在插件管理中选择该文件夹进行安装

> 💡 **提示**：插件无需任何配置，启动后自动在播放器右下角显示信息。

## 🎮 使用说明

播放任意歌曲（本地文件或网络流），播放器右下角将自动显示信息卡片。

### 显示格式示例

| 文件类型 | 显示效果 |
|----------|----------|
| **FLAC 文件** | `44100 Hz · 16bit` + `12.34 MB` |
| **MP3 文件** | `44100 Hz` + `8.50 MB` |
| **未知/不支持** | `未知` |

### 信息说明

- **采样率**：以 Hz 为单位显示（如 44100Hz、48000Hz）
- **位深度**：仅 FLAC 格式支持显示（如 16bit、24bit）
- **文件大小**：自动转换为 B / KB / MB / GB 单位

若歌曲格式非 FLAC/MP3，或数据不完整，则显示"未知"。

---

## ⚠️ 注意事项

1. **跨域支持**：插件依赖浏览器 `fetch API` 发起 Range 请求，请确保 MoeKoe Music 运行在允许跨域的环境（通常 Electron 默认允许）
2. **Range 请求**：仅当歌曲 URL 可访问且服务器支持 `Range` 请求头时，才能正确获取文件大小与头部信息
3. **流媒体限制**：对于某些流媒体或受限 URL，可能无法获取信息，此时会显示"未知"

## 🔧 技术细节

### 解析逻辑

#### FLAC 格式

1. 检查文件头部是否包含 `fLaC` 标记
2. 解析 **STREAMINFO** 元数据块，提取采样率和位深度
3. 采样率：从 STREAMINFO 的第 10-12 字节提取
4. 位深度：从 STREAMINFO 的第 12-13 字节提取，计算方式为 `(raw & 0x1F) + 1`

#### MP3 格式

1. 检查是否存在 **ID3v2** 标签，计算偏移量
2. 跳过 ID3v2 标签后，查找同步帧标记 `0xFFE0`
3. 根据帧头解析采样率索引，映射到对应采样率值

### 更新机制

- 通过 `setInterval` 每 **1 秒** 调用 `updateDisplay()`
- 使用 `playHash` 作为唯一标识，比较当前歌曲是否变化
- 避免重复请求和无效更新
